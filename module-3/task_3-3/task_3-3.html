1. 1. What is the difference between invoking and creating a function?

    2. - Å lage en funksjon vil si å definere hva funksjonen skal gjøre. Å kalle på en funksjon vil si å bruke funksjonen ved å kjøre koden inni den.

  2. Explain the syntax regarding semicolons and code blocks.

    3. - Et semikolon avslutter en instruksjon. En kodeblokk er en samling instruksjoner som hører sammen, og den skrives vanligvis mellom krøllparenteser. Når vi bruker kodeblokker, trenger vi ikke semikolon etter hver linje inni blokken, men ofte etter hele blokken.

  3. What happens when we try to store values from a function when it doesn't return any?

    4. - Hvis en funksjon ikke har en return-setning, returnerer den undefined. Hvis vi prøver å lagre resultatet i en variabel, vil variabelen få verdien undefined.

  4. How do we separate parameters from each other when creating a function?

    5. - Parameterne skilles fra hverandre med ett komma.

  5. Explain how parameter names are related during the invocation and creation of a function.

    6. - Parameternavnene brukes bare inni funksjonen. Når vi kaller funksjonen, spiller ikke navnene på argumentene noen rolle. Det er rekkefølgen som bestemmer hvilken verdi som hører til hvilken parameter.

  6. Explain why a parameter is undefined when a function call is missing a parameter.

    7. - Den er udefinert fordi hvis vi ikke sender med en verdi for en parameter, får parameteren ingen verdi. Da blir den automatisk satt til undefined.

  7. Explain what happens when we use recursive functions.

    8. - En rekursiv funksjon er en funksjon som kaller seg selv. Hver gang funksjonen kalles, opprettes et nytt funksjonskall i minnet. Det er viktig å ha en stoppbetingelse, ellers vil funksjonen fortsette å kalle seg selv og føre til en uendelig løkke eller krasj.
